# Difficultés rencontrées

Une des principales difficultés rencontrées a été la gestion des conflits de fusion (merge) dans GitHub. Lorsque plusieurs branches étaient modifiées simultanément, des conflits sur certains fichiers sont apparus, nécessitant une attention particulière pour s'assurer que le code final soit cohérent.

# Succès

Le code a été divisé avec succès en différentes parties, ce qui améliore considérablement la visibilité et la compréhension. Chaque fichier contient désormais une seule méthode, ce qui rend le projet plus lisible et plus facile à maintenir. L'intégration de la commande `info` a également permis d'ajouter des fonctionnalités utiles, telles que l'affichage de l'heure actuelle, de la version de Java et des informations sur le système d'exploitation.

# Echecs

Un des échecs notables est que, bien que la commande `info` ait été implémentée, elle continue de prendre en compte les paramètres passés derrière elle, au lieu de les ignorer comme prévu.
